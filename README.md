## About

Kraken Plugins is a suite of installable plugins fully compatible with RuneLite that extend your client's capabilities beyond RuneLite's restrictive plugin approval process. Built on top of RuneLite, Kraken enables new overlays for bosses, skilling, and PvP that help you tackle the content you're interested in.

### What Can Kraken Plugins Do?

- üéØ **Boss Mechanics**: Show Olm's current rotation, Sotetseg maze paths, correct tiles for insanity Wardens, projectile splatter locations for Araxxor and so much more!
- ‚öíÔ∏è **Quality of Life**: Large, clickable interfaces for smithing and other skills
- ‚öîÔ∏è **Automation Helpers**: Automatic prayer switching, gear swaps, spell casting, thrall summons, and more
- üé® **Enhanced Overlays**: Visual aids designed to help with in-game tasks rather than fully automate them

Kraken's plugins are mostly overlay-based, providing a middle ground between standard RuneLite plugins and full automation, which drastically reduces ban risk while still offering powerful assistance.

### Contributing

We welcome contributions to our open source components! Here's how you can get involved:

#### Setting Up for Development

1. Clone the relevant repository
2. Check out our [API documentation](https://kraken-plugins.com/docs/api/intro.html)
3. Review the [API Javadocs](https://kraken-plugins.com/javadoc/index.html)

#### Building Your Own Plugins

The Kraken API is open source and available for developers who want to create their own plugins. You can:

- Use the API for custom plugin development
- Create sideloaded or third-party plugins
- Automate tasks using our scripting capabilities

Check out our developer documentation:
- [API Access](https://kraken-plugins.com/docs/api/intro.html)
- [API Usage Guide](https://kraken-plugins.com/docs/api/using.html)
- [Scripting Documentation](https://kraken-plugins.com/docs/api/scripting.html)
- [Building Plugins](https://kraken-plugins.com/docs/api/build.html)

#### Contribution Guidelines

- Fork the repository and create a feature branch
- Follow existing code style and conventions
- Write clear commit messages
- Submit a pull request with a description of your changes
- Ensure all tests pass before submitting
